# Dearest Discord Operating System

Zurg note: this file is the server/channel structure to create manually unless Discord admin API tools are available in the environment.

## Category: Dearest
- `#dearest` — main daily command center
- `#dearest-build` — app build updates, live links, bugs, deploy notes
- `#dearest-design` — logo, UI inspiration, App Store screenshots, brand system
- `#dearest-security` — PIN editing, encryption, legacy contacts, abuse prevention, privacy
- `#dearest-ideas` — feature backlog, wild ideas, user stories, future versions
- `#dearest-b2c-appstore` — consumer App Store strategy, pricing, ASO, screenshots, paywall
- `#dearest-b2b-hospitals` — hospital/hospice/palliative care strategy and sales material
- `#dearest-reports` — finalized reports, pitch docs, investor/admin summaries

## Suggested channel topics
**#dearest-build:** Live app: https://zurg888.github.io/dearest-app/ — prototype repo: https://github.com/Zurg888/dearest-app

**#dearest-design:** Warm Apple-like UI: ivory, parchment, mauve ribbon, dusty rose, gold glow, navy privacy accents. Avoid wedding/memorial-only vibes.

**#dearest-security:** Private by default. PIN/Face ID for edits. Edit history. Export anytime. No ads. No AI training on private memories.

**#dearest-b2b-hospitals:** B2B strategy for hospitals, hospice, palliative care, oncology, patient experience, chaplaincy, and bereavement programs.

**#dearest-b2c-appstore:** Consumer strategy for App Store: family memories, legacy letters, voice notes, subscriptions, retention, ASO.
