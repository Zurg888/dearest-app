# Dearly Discord Operating System

Zurg note: this file is the server/channel structure to create manually unless Discord admin API tools are available in the environment.

## Category: Dearly
- `#dearly` — main daily command center
- `#dearly-build` — app build updates, live links, bugs, deploy notes
- `#dearly-design` — logo, UI inspiration, App Store screenshots, brand system
- `#dearly-security` — PIN editing, encryption, legacy contacts, abuse prevention, privacy
- `#dearly-ideas` — feature backlog, wild ideas, user stories, future versions
- `#dearly-b2c-appstore` — consumer App Store strategy, pricing, ASO, screenshots, paywall
- `#dearly-b2b-hospitals` — hospital/hospice/palliative care strategy and sales material
- `#dearly-reports` — finalized reports, pitch docs, investor/admin summaries

## Suggested channel topics
**#dearly-build:** Live app: https://zurg888.github.io/dearly-app/ — prototype repo: https://github.com/Zurg888/dearly-app

**#dearly-design:** Warm Apple-like UI: ivory, parchment, mauve ribbon, dusty rose, gold glow, navy privacy accents. Avoid wedding/memorial-only vibes.

**#dearly-security:** Private by default. PIN/Face ID for edits. Edit history. Export anytime. No ads. No AI training on private memories.

**#dearly-b2b-hospitals:** B2B strategy for hospitals, hospice, palliative care, oncology, patient experience, chaplaincy, and bereavement programs.

**#dearly-b2c-appstore:** Consumer strategy for App Store: family memories, legacy letters, voice notes, subscriptions, retention, ASO.
